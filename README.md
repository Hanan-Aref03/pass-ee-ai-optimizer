# PASS EE Optimizer

This repository now has two clear lanes:

- `matlab/legacy/` contains the corrected MATLAB simulation and dataset-generation bundle.
- `ml/pass_dnn/` will hold the DNN surrogate implementation for PASS dataset training and inference.

## What lives where

- `matlab/legacy/`
  - Original MATLAB optimization, channel, rate, and visualization scripts
  - Dataset-generation scripts
  - Run guides and thesis notes
- `data/templates/`
  - CSV headers that define the DNN input/output schema
- `data/raw/`
  - QoS-labeled PASS CSV pairs, plus any orphaned exports clearly marked as such
- `data/processed/`
  - Merged corpus exports such as `pass_merged_corpus_input.csv` and `pass_merged_corpus_output.csv`
- `ml/pass_dnn/`
  - Tabular dataset loader
  - DNN model
  - Training, evaluation, and inference entry points
- `docs/`
  - Human-readable project layout and DNN design notes

## MATLAB workflow

Use the scripts in `matlab/legacy/` exactly as the original bundle intended:

- `main_pass.m` for thesis-style simulation runs
- `generate_pass_dataset.m` for PASS training data
- `visualize_results.m` for saved `.mat` result files
- `data.m` to export a saved result file to CSV

The dataset templates under `data/templates/` match the CSV schema produced by the MATLAB scripts.

## DNN starter workflow

The initial surrogate model is designed for PASS data:

- Inputs: 3 user positions plus QoS (`7` features total)
- Outputs: 9 PA x-positions plus 3 waveguide powers (`12` targets total)

The trainer now adds a few higher-value engineering layers:

- combines every complete PASS pair in `data/raw/`
- ignores incomplete exports
- augments training rows using 3-user / 3-waveguide symmetry
- balances training batches by QoS frequency by default
- uses a robust regression loss by default
- stores reproducibility metadata and XAI outputs

Start with:

```bash
python scripts/audit_pass_raw_data.py
python scripts/train_pass_dnn.py
python scripts/evaluate_pass_dnn.py --artifact-dir ml/pass_dnn/artifacts/<run-folder>
python scripts/explain_pass_dnn.py --artifact-dir ml/pass_dnn/artifacts/<run-folder>
python scripts/pass_dnn_gui.py --artifact-dir ml/pass_dnn/artifacts/<run-folder>
python scripts/build_pass_corpus.py
```

You can also inspect the package entry point with:

```bash
python -m ml.pass_dnn --help
```

Once a PASS dataset exists, train with the input/output CSV pair generated by `generate_pass_dataset.m`.

## How to read the results

- This is a regression surrogate, so `MAE`, `RMSE`, and `R^2` are the main metrics.
- `accuracy` and `F1` are not the right primary scores here because the current dataset has no meaningful negative class.
- For system behavior, watch:
  - `qos_satisfaction_rate_pred`
  - `sum_rate_mae`
  - `ee_mae`
  - `feasible_power_rate_projected`
- If you want a binary feasibility check, use:
  - `qos_accuracy`
  - `qos_f1`
  - `qos_balanced_accuracy`

## More MATLAB data

Your current raw PASS corpus now spans multiple QoS values and is merged automatically during training. The training loop also balances QoS batches so `0.1`, `0.3`, `0.5`, and `0.7` all contribute.

For any next MATLAB generation pass, keep the same sweep pattern or extend it with more intermediate QoS values if you want finer operating-point coverage:

- `QoS_R = 0.3`
- `QoS_R = 0.5`
- `QoS_R = 0.7`

That matches the project guidance in `matlab/legacy/DATASET_GENERATION.txt` and gives the DNN a much better chance to generalize across operating points.
