Two simulation modes (selected by param.rand_sc in main_pass.m)
-------------------------------------------------------------
1) Conventional fixed BS antennas:
   param.rand_sc = 0;

   - No waveguides. Nwg=3 is used only to set Na=3 fixed BS antennas.
   - Three fixed antenna positions in fmincon_fpa.m (vertical ULA at x=0, spacing lambda/2 along y).
   - Only transmit power is optimized; fmincon_fpa finds [power_ant1, power_ant2, power_ant3].
   - CSV output: optimized powers only.

2) PASS (Pinching Antenna System):
   param.rand_sc = 1;

   - Three waveguides with three pinching antennas (PAs) on each.
   - PA x-positions and waveguide powers are jointly optimized by fmincon_pass.
   - CSV output: 9 PA x-positions + 3 waveguide powers.

Note: generate_random_pass_positions.m is used only as random starting points inside
fmincon_pass (10 initializations). It is not a separate simulation mode.

Important Modeling Change
-------------------------
The effective waveguide-index parameter was removed from all MATLAB files. The waveguide
model uses phase rotation only with the free-space wavelength and does not include
waveguide path loss.

Visualization
-------------
Run visualize_results.m and select a MAT file generated by main_pass.m. The script creates
five figures and saves both .fig and .png copies in figures_from_visualize_results next
to the selected MAT file.

ML dataset for DNN (PASS surrogate)
---------------------------------
Run generate_pass_dataset.m to collect many labeled PASS samples efficiently.
See DATASET_GENERATION.txt for workflow, speed tips, and train/test split advice.
